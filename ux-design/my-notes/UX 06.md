# 1. Prototyping in UX Design
Prototyping is the process of creating a **working model of a digital product** to demonstrate how the final application will function.

Purpose:
- visualize the product before development
- test ideas quickly
- present the concept to stakeholders and users
- identify design problems early.

A prototype is **not the final product**, but a simulation of how the product will behave.

---

# 2. Role of Prototypes in the UX Process
Prototypes are created after planning stages such as:

- user research
- user flows
- wireframes.

They help translate design ideas into **interactive experiences** that stakeholders and users can explore.

Benefits:
- reduces development risk
- allows quick design changes
- improves communication with stakeholders.

---

# 4. Figma Environment Overview
The Figma interface consists of several main areas.

### Navigation panel
Contains:
- recents
- drafts
- teams
- community resources.

Used for organizing design files.

---

### Design canvas
The main workspace where screens and UI elements are created.

Supports:
- zooming
- panning
- arranging design elements.

---

### Toolbar
Provides tools for creating elements such as:
- frames
- shapes
- text
- vectors
- images.

---

### Layers panel
Displays all elements placed on the canvas.

Used for:
- selecting objects
- organizing components
- managing hierarchy.

---

### Properties panel
Allows editing properties of selected objects.

Examples:
- color
- size
- positioning
- border radius.

---

# 5. Creating a New Design File
To begin prototyping in Figma:

1. Open **figma.com**
2. Select **New Design File**
3. A new canvas workspace is created.

This workspace will contain the screens for the prototype.

---

# 6. Frames in Figma
A **frame** represents a screen in the application.

Frames are used to simulate devices such as:
- mobile phones
- tablets
- desktops.

Examples of available presets:
- Android devices
- iPhone devices
- desktop screens.

Each screen in the app is designed inside its own frame.

---

# 7. Example Prototype Project
The lesson demonstrates building a prototype for a simple plant information app called **Planto**.

The app provides:
- plant categories
- plant species information
- plant care details
- augmented reality visualization.

The first stage of the prototype focuses on creating **two screens**.

---

# 8. Screen 1 — Splash Screen
The splash screen is the **entry point** of the application.

Elements created:

### Background layer
A rectangle covering the entire frame used as the screen background.

### Color palette
Colors are selected using a color palette tool.

Color palettes are defined using **hex codes**.

Example:
Primary color applied to the background.

---

### Icon
An icon representing the app is added.

Source:
Google Material Icons.

Preferred file format:
**SVG**

Advantages of SVG:
- scalable without losing quality
- editable colors
- vector-based graphics.

---

### Application logo text
Text added using the text tool.

Example:
App name “Planto”.

Font used:
Roboto Slab.

Text alignment ensures the logo is **perfectly centered on the screen**.

---

# 9. Color Palette Selection
Color palettes help maintain **consistent visual identity**.

Example tool used:
Material palette generator.

Palette components include:
- primary color
- light primary color
- dark primary color
- accent color.

Hex codes allow designers to apply colors consistently across the interface.

# 10. Screen 2 — Home Screen
The second screen represents the **main entry interface of the app**.

This screen is created by duplicating the splash screen frame and modifying it.

Key elements include:

### Background transition design
A curved design created using:
- ellipse shape
- rectangle shape
- boolean subtraction.

This creates a visual transition between screens.

---

### Logo and icon repositioning
The app logo and icon move to the top section of the screen.

---

### Call-to-action button
A **Get Started** button is added.

Button components:
- rectangle shape
- text label.

Button design improvements:
- rounded corners
- centered text
- consistent color styling.

---

# 11. Button Design Principles
Buttons are designed with several UX considerations.

Elements include:
- color contrast
- rounded corners
- clear label text.

Rounded corners improve modern UI appearance.

Example setting:
Border radius applied to soften edges.

---

# 12. Grouping Elements in Figma
Multiple elements can be grouped together.

Purpose:
- move related components together
- simplify layer management.

Example grouping:
- button shape
- button text.

Grouped elements behave as a single component.

---

# 13. Adding Interactions in Prototypes
Interactions simulate how users interact with the interface.

Example interactions include:
- tapping
- scrolling
- dragging.

In this lesson, the prototype interaction is:

**Tap → Navigate to another screen**

---

# 14. Navigation Between Screens
To create navigation:

1. Select the source screen
2. Open the **Prototype tab**
3. Add interaction rule
4. Define action:
   - On tap
   - Navigate to destination screen.

This creates a visual connection between screens.

---

# 15. Smart Animate
Figma provides animation effects for transitions.

Example used:
**Smart Animate**

Purpose:
- smoothly animate changes between screens
- create natural movement between elements.

Animation properties include:
- easing style
- duration.

Example duration:
300 milliseconds.

---

# 16. Testing the Prototype
Figma allows designers to run the prototype.

Process:
- click **Present / Play**
- the interface runs as an interactive simulation.

Users can click through screens and experience the interaction flow.

---

# 17. Advantages of Prototyping with Figma
Figma provides several benefits for UX designers.

Advantages include:
- cloud collaboration
- real-time editing
- easy sharing with stakeholders
- quick iteration of designs.

Prototypes help teams validate ideas **before development begins**.

---

# 18. Relationship Between User Flow and Prototyping
User flows define the **logical path of interaction**.

Prototypes convert that flow into **interactive screens**.

Process sequence:

User Flow → UI Design → Prototype → User Testing.

The prototype visually implements the navigation structure defined earlier.

---

# 19. Key Takeaways
Important concepts to remember:

- Prototyping creates interactive representations of digital products.
- Figma is a cloud-based tool used for UI design and prototyping.
- Frames represent individual screens of an application.
- SVG icons are preferred because they are scalable and editable.
- Color palettes help maintain consistent visual design.
- UI elements include backgrounds, icons, text, and buttons.
- Interactions define how users move between screens.
- Smart animation improves transitions between interface states.
- Prototypes allow testing and feedback before development begins.
