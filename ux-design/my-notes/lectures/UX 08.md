# Completing the UX Prototype (Planto App)

This lesson completes the **first full prototype** built in Figma.  
The final tasks include:

- building **plant detail screens**
- implementing the **remaining user flows**
- adding an **AR (Augmented Reality) feature**
- implementing **back navigation**

At the end of this lesson, the entire application flow becomes functional.

---

# 1. Final Application Structure

By the end of the prototype, the application contains these screens:

1. Splash Screen  
2. Home Screen  
3. Category Screen  
4. Plant Type Screen  
5. Plant Detail Screens  
6. AR Mode Screen  

Navigation flow:

```

Splash → Home → Categories → Plant Type → Plant Detail → AR Mode

```

Back navigation allows users to move backwards through the flow.

---

# 2. Creating the Plant Detail Screen

A **plant detail screen** displays specific information about an individual plant species.

Example plant:

```

Agave (Shark Skin Succulent)

```

## Step 1 — Create a Frame

Create a new frame using:

```

Frame → Android Small

```

Rename frame:

```

agave plant

```

---

# 3. Plant Image Section

Add a rectangle container to hold the plant image.

Example dimensions:

```

325 × 254 px

```

Then insert the plant image using:

```

File → Place Image

```

Example image used:

```

Agave

```

This large image acts as the **hero image** of the detail screen.

---

# 4. Plant Title

Add the plant name below the image.

Example:

```

Agave Shark Skin

```

Formatting:

- Font: Source Sans Pro
- Font size: 20
- Style: Bold
- Color: Dark green theme

Purpose:

The title clearly identifies the plant species.

---

# 5. Plant Information Card

Below the plant title, a **white information card** is created to display plant attributes.

Rectangle dimensions:

```

328 × 189 px

```

Design styling:

- rounded corners
- drop shadow
- white background

Corner radius:

```

15 px

```

Drop shadow opacity:

```

35%

```

This creates a clean **content container**.

---

# 6. Plant Attribute Icons

Icons are imported from **Google Material Icons**.

Three icons are used:

1. Description icon  
2. Water drop icon  
3. Sun / light icon  

Icons represent key plant attributes.

Example mapping:

| Icon | Attribute |
|-----|-----|
| Document | Type |
| Water drop | Moisture |
| Sun | Light |

Icons are resized to around:

```

~31–35 px

```

---

# 7. Plant Attribute Labels

Text labels are added next to each icon.

Example labels:

```

Type
Moisture
Light

```

Font settings:

- Font: Source Sans Pro
- Size: 16
- Color: Green theme

---

# 8. Plant Attribute Values

Descriptions are added below each label.

Example values for **Agave**:

```

Type: Succulent
Moisture: Drought tolerant
Light: Full sun

```

Font settings:

```

Font size: 12
Style: Regular

```

This layout creates a simple **information list**.

---

# 9. Adding the AR Button

A button is added at the bottom of the screen.

Text:

```

View in Room

```

Purpose:

This button opens **Augmented Reality Mode** to visualize the plant.

The button is reused from earlier designs to maintain UI consistency.

---

# 10. Duplicating Plant Detail Screens

Instead of redesigning screens, the Agave screen is duplicated.

Shortcut:

```

Ctrl + D

```

Two additional screens are created:

1. **Black Rose Plant**
2. **Red Aloe Plant**

Changes required:

- update plant name
- replace plant image
- adjust moisture description if needed

Example:

Black Rose:

```

Moisture: Dry to medium
Light: Full sun

```

---

# 11. Connecting Plant Type → Plant Detail

Prototype interactions are added.

Example:

```

On Tap → Navigate To → Agave Plant Screen

```

Similar flows are created for:

- Black Rose
- Red Aloe

This connects the plant list to the detailed view.

---

# 12. Creating AR Mode Screen

A new frame is created.

Name:

```

AR Mode

```

Basic elements include:

- back arrow
- title
- animated visual

---

# 13. Adding AR Simulation

Figma **does not support video formats** like:

- MP4
- MKV
- AVI

Instead, motion is simulated using:

```

GIF files

```

GIFs are treated as images but display animated frames.

Steps:

```

File → Place Image → Insert GIF

```

The GIF represents the plant being viewed in **Augmented Reality mode**.

---

# 14. Adjusting AR Media

If needed:

- rotate the image
- scale it
- send it behind UI layers

Rotation shortcut:

```

Ctrl + Shift + Drag

```

The back arrow remains on top for navigation.

---

# 15. Connecting AR Button

Interaction for the button:

```

On Tap → Navigate To → AR Mode Screen

```

Animation:

```

Slide In

```

This makes the AR screen appear naturally during navigation.

---

# 16. Implementing Back Navigation

Back arrows are added across screens.

Interaction setting:

```

On Tap → Back

```

The **Back action** returns the user to the previous screen in the navigation stack.

This is applied to:

- plant detail screens
- AR screen

---

# 19. Prototyping Tip for UX Designers

Even if functionality is not implemented, prototypes should **demonstrate potential features**.

Examples:

- AR product visualization
- plant scanning
- AI recommendations

This helps communicate product vision to:

- stakeholders
- investors
- developers
- users

---

# 20. Important UX Lessons From This Prototype

### 1. Consistent design system
Reusing components maintains visual consistency.

### 2. Structured naming
Proper naming improves collaboration with developers.

### 3. Interaction feedback
Animations and transitions improve usability.

### 4. Modular design
Duplicating components saves time.

### 5. Feature demonstration
Prototypes can showcase features that don't yet exist.
