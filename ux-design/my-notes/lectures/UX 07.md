# Building Additional Screens & Implementing User Flows in Figma

## 1. Lesson Focus
This lesson continues the development of the **Planto mobile prototype** inside Figma.

Main focus:
- working with **elements**
- manipulating **element properties**
- implementing **user flows and navigation**
- building additional **application screens**

Two major screens are built in this lesson:
1. **Category Screen**
2. **Plant Type Screen**

---

# 2. Application Flow Context
From the earlier user flow diagram, the application contains these screens:

1. Splash Screen
2. Home Screen
3. Category Screen
4. Plant Type Screen
5. Plant Details Screen
6. AR Mode Screen

This lesson focuses on creating:

**Category Screen → Plant Type Screen**

# 3. Creating the Category Screen

## Step 1 — Create a Frame
Create a new mobile frame.

Frame preset:
Android Small

Rename frame:
```

category screen

```

This screen will display the different plant categories.

---

# 4. Adding Navigation Elements

## Back Arrow
A back arrow icon is imported from **Google Material Icons**.

Steps:
1. Search for **arrow back**
2. Download as **SVG**
3. Import into Figma using:

```

File → Place Image

```

SVG format is used because:
- it scales without losing quality
- colors can be modified

---

## Page Title
A heading is added next to the back button.

Text:
```

Explore

```

Purpose:
This title communicates that users are exploring plant categories.

Font:
- Source Sans Pro
- Size: 30px

---

# 5. Creating Category Cards

The category screen displays **three category cards**:

1. Succulents
2. Vegetables
3. House Plants

Each category is represented as a **visual card**.

---

# 6. Designing a Category Card

## Card Container
Create a rectangle.

Dimensions:
```

355 × 170 px

```

This rectangle acts as the **card holder**.

---

## Adding Image
Insert a plant image representing the category.

Example:
```

succulents image

```

Steps:
- Place image
- Scale using Shift
- Use **crop** to fit inside the card

---

## Dark Overlay Mask
A semi-transparent mask is placed on top of the image.

Color:
```

#282727

```

Opacity:
```

40%

```

Purpose:
- improves text visibility
- creates a modern UI style

---

## Animated Vertical Line
A vertical colored line is added.

Line width:
```

5px

```

Color:
dark green

Purpose:
Used later for **hover animation feedback**.

---

## Category Title
Add the category name.

Example:
```

Succulents

```

Text style:
- SemiBold
- Size: 24px

This visually distinguishes it from the main page heading.

---

# 7. Organizing UI Elements

Each element inside the card is renamed:

Examples:

```

succulents line
succulents mask
succulents image
succulents holder

```

Then grouped as:

```

succulents card

```

Proper naming helps maintain **clean project structure**.

---

# 8. Reusing UI Components

Instead of designing new cards from scratch:

Use duplication.

Shortcut:
```

Ctrl + D

```

The first card is duplicated twice to create:

- Vegetables card
- House plants card

Images and text are then replaced accordingly.

---

# 9. Adding Depth with Shadows

To make the UI feel more realistic:

Apply **Drop Shadow**.

Opacity:
```

35%

```

Benefits:
- adds visual depth
- separates cards from background

---

# 10. Linking Screens (Navigation Flow)

Navigation is implemented using Figma's **Prototype tab**.

Example:

Interaction:
```

On Tap → Navigate To → Category Screen

```

Animation:
```

Slide In

```

```

Home Screen → Category Screen

```

# 11. Creating Hover Animation

To provide interaction feedback:

A second version of the category screen is created.

Duplicate screen:
```

category screen → category screen anim

```

Changes in animated version:
- vertical line expands
- image zooms slightly

---

## Animation Changes

### Line Expansion
Line width increased from:
```

5px → 12px

```

### Image Zoom
Image is scaled slightly larger and recropped.

Purpose:
Create **visual feedback** when user interacts.

---

# 12. Triggering Animation

Interaction settings:

```

While Hovering → Open Overlay

```

Effect:
```

Dissolve

```

This produces a smooth hover animation effect.

# 23. Key UX Concepts Demonstrated

Important UX principles shown in this lesson:

### 1. Reusable Components
Reuse UI modules instead of redesigning them.

### 2. Visual Feedback
Hover animations indicate interaction.

### 3. Hierarchy
Different text sizes distinguish headings and subheadings.

### 4. Clean UI Structure
Proper naming and grouping improve workflow.

### 5. Rapid Prototyping
Design features even if functionality is not implemented.
